# ASCII文本化UI布局描述通用提示词

## 核心要求

请用**纯文本ASCII字符**精确描述界面布局，让阅读者能够直观理解设计结构，就像看到线框图一样清晰。

## 描述规范

### 1. 使用ASCII边框字符绘制结构

用以下字符绘制布局框架：

- 边框角：`┌` `┐` `└` `┘`
- 横线：`─` `━` (细线用`─`，粗线用`━`)
- 竖线：`│` `┃` (细线用`│`，粗线用`┃`)
- 分隔线：`─────` 或 `━━━━━`

### 2. 层级化描述布局

按照以下结构组织内容：

**【整体布局】**  
简要说明画布属性和布局方式

**【顶部/头部区域】**  

```
┌────────────────────────────────────┐
│ 用文字和符号描述这里的内容          │
└────────────────────────────────────┘
```

**【主体区域】**  
说明主体采用的布局方式（网格/分栏/列表等），然后逐个描述每个区块：

```
【区块1：位置说明 - 功能名称】
┌─────────────────────────────┐
│ 标题/图标                    │
│ ─────────────────────────── │
│                             │
│ 具体内容描述                 │
│ • 列表项                     │
│ • 列表项                     │
│                             │
└─────────────────────────────┘
```

**【底部区域】**  
同样用边框描述

**【视觉规范】**  
列出颜色、字号、圆角、间距等具体数值

### 3. 灵活运用符号

- 列表：`□` `•` `○` `▪` `-`
- 图标占位：`🎯` `📁` `⚡` 或 `[图标]`
- 强调：数字、关键词可标注颜色说明
- 进度条：`━━━━━━░░░░` 或用文字说明
- 分组：用空行和分隔线区分

### 4. 保持直观性

- 边框宽度反映实际占比关系
- 用缩进表达层级关系
- 关键信息用符号或留白突出
- 复杂布局可拆分多层描述

## 输出目标

让读者**无需看到图片**就能在脑海中准确还原出界面的：

1. 整体结构和布局方式
2. 每个区域的位置和大小关系
3. 内容的层级和组织方式
4. 视觉风格和设计规范

## 示例参考

当需要描述一个看板时：

- 先用文字说明"2x2网格布局"
- 再用ASCII边框画出每个卡片的位置
- 在边框内用文字和符号描述卡片内容
- 最后补充视觉规范细节

关键：**形式服从内容**，根据实际需求选择最直观的ASCII表达方式。
